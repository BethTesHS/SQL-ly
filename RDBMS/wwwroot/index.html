SQLly Command Tutorial

Below is a guide to the SQLly commands. Copy and paste these into the Console to try them out.

1. Creating, Managing and Dropping Tables (Primary & Unique Keys)

Primary Keys: Every table must have a column named id of type int. This acts as the Primary Key and must be unique.
Unique Constraints: Add the UNIQUE keyword after a type to enforce uniqueness on other columns (like emails or usernames).

-- Syntax: CREATE TABLE [name] (col1 type [UNIQUE], col2 type) CREATE TABLE users (id int, username string UNIQUE, age int) -- Delete a table DROP TABLE users

2. Create (Insert)

Add new records. The system will block duplicate IDs or duplicate values in UNIQUE columns.

INSERT INTO users VALUES (1, "Alice", 25) INSERT INTO users VALUES (2, "Bob", 30) -- This will fail because "Alice" exists: INSERT INTO users VALUES (3, "Alice", 40)

3. Read (Select & Join)

View data. You can join any two tables on any two columns.

-- Simple Select SELECT * FROM users -- Filter by ID SELECT * FROM users WHERE id=1 -- Dynamic Join (Syntax: t1 JOIN t2 ON t1.c = t2.c) SELECT * FROM users JOIN orders ON users.id = orders.user_id

4. Update

Modify existing records by ID. Unique constraints are checked during updates.

UPDATE users SET age=26 WHERE id=1

5. Delete

Remove specific records by their ID.

DELETE FROM users WHERE id=1

6. Joining Tables

Combine rows from two tables based on a related column.
SQLly performs an Inner Join, meaning only rows that have matching values in both tables will be displayed.

Syntax Rule: You must specify the table name for the columns in the condition (e.g., table.column).

-- Syntax: SELECT * FROM [t1] JOIN [t2] ON [t1.col] = [t2.col] SELECT * FROM users JOIN orders ON users.id = orders.user_id
